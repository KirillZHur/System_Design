# 4. Основные задачи пользователей и их реализация

## Задачи пользователей

### Создание нового пользователя
- Реализовано через Web UI и API Gateway.
- API Gateway направляет запрос в User Service.
- User Service сохраняет данные пользователя в базе данных.
- После регистрации система отправляет уведомление через внешний почтовый сервис.

### Поиск пользователя по логину или маске имени и фамилии
- Реализовано через API Gateway и User Service.
- Поиск осуществляется в базе данных пользователей.

### Создание курса
- Реализовано через Web UI и API Gateway.
- API Gateway направляет запрос в Course Service.
- Course Service сохраняет информацию о курсе в базе данных.

### Получение списка курсов
- Реализовано через API Gateway и Course Service.
- Course Service получает данные из базы данных.

### Добавление урока в курс
- Реализовано через API Gateway и Course Service.
- Course Service сохраняет данные урока в базе данных.

### Получение уроков курса
- Реализовано через API Gateway и Course Service.
- Course Service получает список уроков из базы данных.

### Запись пользователя на курс
- Реализовано через API Gateway и Enrollment Service.
- Enrollment Service сохраняет информацию о записи в базе данных.

### Получение курсов пользователя
- Реализовано через API Gateway и Enrollment Service.
- Enrollment Service получает данные из базы данных.

### Отметка о прохождении урока
- Реализовано через API Gateway и Enrollment Service.
- Enrollment Service обновляет информацию о прогрессе обучения в базе данных.

# 5. Перечень контейнеров

Контейнеры системы:

### **Web UI**
- Клиентское веб-приложение системы.
- Обеспечивает пользовательский интерфейс для гостей, студентов, преподавателей и администраторов.
- Отправляет HTTPS-запросы к API Gateway.
- Технологии: Angular (SPA).

### **API Gateway**
- Точка входа для всех клиентских запросов.
- Выполняет маршрутизацию запросов к соответствующим сервисам.
- Технологии: HTTP/HTTPS API Gateway (NGINX / Kong).

### **User Service**
- Управляет данными пользователей системы.
- Реализует операции регистрации и поиска пользователей.
- Технологии: C++, userver framework.

### **Course Service**
- Управляет курсами и уроками.
- Реализует создание курсов, добавление уроков и получение информации о курсах.
- Технологии: C++, userver framework.

### **Enrollment Service**
- Управляет записями пользователей на курсы и прогрессом обучения.
- Реализует регистрацию пользователя на курс, получение курсов пользователя и отметку о прохождении уроков.
- Технологии: C++, userver framework.

### **База данных LMS**
- Хранит данные пользователей, курсов, уроков, записей на курсы и прогресса прохождения.
- Технологии: PostgreSQL.

### **Почтовый сервис**
- Внешняя система отправки email уведомлений.
- Используется для подтверждения регистрации и отправки уведомлений.
- Технологии: SMTP.

## Технологический стек

- **C++ / userver**

  - Enrollment Service

# 9. Диаграмма Dynamic

## Диаграмма Dynamic

Диаграмма показывает процесс записи пользователя на курс:

- **Студент → Web Application → API Gateway → Enrollment Service**
- **Enrollment Service → Database → Сохранение записи пользователя на курс**
- **Студент → Web Application → API Gateway → Course Service**
- **Course Service → Database → Получение информации о курсе**
